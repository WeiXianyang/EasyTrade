# EasyTrade 商城与后台体验改造设计

日期：2026-06-09

## 背景

本次改造面向课程演示，优先保证效果清楚、链路稳定、工程改动可控。现有项目是 Vite + React 单入口应用，商城与后台都挂在同一路由树中，商城底部导航包含后台入口，购物车主要通过浮窗抽屉进入。用户希望后台像另一套前端、购物车进入主导航、浮窗改成智能客服，并补齐购物车选择、分类筛选、首页动效、骨架屏以及“我的”页的收藏关注足迹模块。

## 目标

- 商城与后台在开发演示时监听不同端口，呈现为两套前端。
- 登录页提供“用户 / 管理员”身份选择，课程演示时保留账号提示，降低演示成本。
- 商城导航改为“首页 / 购物车 / 分类 / 我的”，购物车位于第二个位置，后台不再出现在商城导航中。
- 原购物车浮窗改成智能客服浮窗，支持简单问答和商品推荐链接。
- 购物车支持单选、全选、部分选择、取消全选，结算仅使用已选商品。
- 分类页增加右上角筛选/排序能力。
- 首页增加可见的 transform 动效、主题切换过渡、限时促销/秒杀价浮动效果和加载骨架屏。
- “我的”页增加收藏、关注、足迹等常见购物平台入口。

## 推荐方案

采用“双入口双端口”的课程演示方案：

- 商城入口继续使用 `index.html` 与 `src/main.jsx`。
- 后台新增 `admin.html` 与 `src/admin-main.jsx`，复用现有 `AdminLayout`、后台页面、权限守卫、AppProvider 和 localStorage 数据服务。
- 新增开发脚本，例如 `dev:shop` 与 `dev:admin`，分别指定不同端口。`dev` 可继续保留为商城默认入口，降低原有使用成本。
- 构建层面使用 Vite 多页面输入，输出商城和后台两个入口页面。课程演示时也可以分别打开两个端口说明“商城端/后台端分离”。

该方案比完全拆目录更轻，比单路由更符合“监听不同端口”的需求。

## 页面与组件设计

### 登录

前台登录页改为身份分段选择：

- 用户模式：保留登录和注册，登录成功后回到来源页或首页。
- 管理员模式：只显示后台账号登录，不提供注册；登录成功后跳转后台端。课程演示保留 `admin/admin123`、`operator/operator123` 提示。
- 独立后台登录页可以保留为后台端直接访问时的兜底页面，也可以复用同一登录组件。

### 商城导航与客服

- `ShopLayout` 的导航项调整为：首页、购物车、分类、我的。
- `selectedKey` 需要识别 `/cart`、`/checkout`、`/pay` 为购物车相关状态。
- 移除商城导航中的后台入口。
- 删除购物车浮窗入口，新增智能客服浮窗组件。浮窗固定在原位置，打开后使用抽屉或小面板展示聊天记录、输入框和推荐链接。
- 客服服务通过 `chatService` 封装。配置读取本地环境变量，不把密钥提交到源码。当前项目为前端课程演示，密钥只用于本地演示；生产环境应改为后端代理。

### 智能客服

客服支持两层逻辑：

- 优先用本地商品数据做简单意图兜底。例如“提亮肤色面膜”没有对应商品时，引导到分类页或推荐相近热卖商品。
- 当环境变量配置完整时，调用自定义 OpenAI 兼容接口，模型返回简短回答。前端将可购买商品名渲染成 `/detail/:productId` 链接。

客服失败时不阻塞购物流程，显示“客服暂时繁忙，可以先看看热卖商品”并提供分类页入口。

### 购物车

现有 `cartService` 已包含 `selected`、`setSelected`、`setAllSelected`、`getSelectedItems` 和 `getSelectedSummary`，保留这套数据模型。

改造重点：

- 购物车页继续使用表格选择列。
- 购物车抽屉也补上每项勾选框和顶部全选/取消全选。
- 加入购物车后默认选中新商品，符合“刚加入即可购买”的电商习惯；用户可以随时取消勾选。
- 结算页继续只读取已选商品。若没有已选商品，按钮禁用并提示用户先选择。

### 分类筛选

分类页在当前分类标题右侧新增筛选控件：

- 排序：默认、价格升序、价格降序、销量优先、折扣优先。
- 筛选：仅看有优惠、库存充足。
- 筛选状态与当前分类一起驱动商品列表，保持纯前端本地计算。
- 移动端使用抽屉或下拉菜单，桌面端使用紧凑按钮和菜单。

### 首页动效与骨架屏

- 首页 hero、分类快捷入口、热门商品卡片增加更明显但克制的 `transform` 入场和悬停效果。
- 主题切换按钮和页面根节点增加过渡类，切换时产生可见的缩放/旋转或色彩过渡。
- 新增“限时促销 / 秒杀价”模块，从已有有折扣商品中取 3 到 4 个商品，价格标签或“秒杀价”文字上下浮动。
- 路由级加载从单一 `Spin` 改为商城骨架屏，包含搜索条、横幅和商品卡骨架；后台可继续使用管理页骨架或 Spin。

### 我的页面

“我的”页新增三个轻量窗口：

- 收藏：用户收藏的商品，支持跳转商品详情。
- 关注：关注的分类或店铺式模块，课程演示可用分类作为关注对象。
- 足迹：最近浏览商品，商品详情页访问时写入 localStorage。

这些数据用新服务封装，例如 `userActivityService`，与订单、购物车一样按用户隔离。

## 数据流

- `AppProvider` 继续负责登录态、主题、购物车数量和全局刷新。
- 商城端和后台端复用同一个 localStorage 命名空间，因此后台商品上下架、价格修改仍能联动前台。
- 客服读取商品服务获取可推荐商品，再根据用户问题和接口返回生成展示消息。
- 足迹由商品详情页写入，收藏可由商品卡或详情页触发，关注由“我的”页或分类页触发。

## 错误处理

- 后台端未登录访问时跳转后台登录。
- 商城端需要用户态的页面仍跳转用户登录。
- 客服接口缺配置、超时或失败时显示本地兜底推荐。
- 购物车中商品下架或库存变化时沿用现有实时商品数据校验，无法购买的商品不进入结算。
- 分类筛选无结果时显示空状态和清除筛选入口。

## 测试计划

- 更新导航静态测试：购物车为第二个导航项，后台不在商城导航中，客服浮窗存在。
- 更新 Vite/脚本测试：存在商城和后台开发脚本，后台入口独立。
- 扩展购物车服务/页面测试：单选、全选、取消全选、结算只统计已选商品。
- 扩展分类页测试：价格排序、销量排序、折扣筛选关键代码存在。
- 扩展用户活动服务测试：收藏、关注、足迹按用户读写并去重。
- 扩展首页 UI 测试：秒杀模块、骨架屏和主题切换动效类存在。
- 完成后运行 `npm test`、`npm run lint`、`npm run build`。

## 不做事项

- 不引入真实后端数据库。
- 不做复杂客服会话管理、人工客服转接或支付级安全能力。
- 不把 API 密钥提交到仓库。
- 不大规模重写现有后台页面和权限体系。
